Quiz 2

Hard to Achieve

Selama mengerjakan tugas phase 2 hal yang sulit (hard) bagi saya untuk diimplementasikan adalah sulitnya melakukan Override pada Materialize, menyimpan data ketika user sudah login, membuat element Sticky.

Sulitnya melakukan Override. Ketika melakukan implementasi hal pertama yang saya lakukan membuat navbar yang sesuai dengan mockup yang sudah saya buat sebelumnya yakni konten pada navbar harus diletakkan pada tengah navbar dengan lebar 50% serta berubah menjadi "Burger-Button" ketika dibuka melalui handphone. Awalnya saya mengira implementasinya akan mudah dengan menggunakan kelas Row dan Col pada Materialize. Pada kenyataannya hal tersebut tidak mudah, karena secara default materialize memiliki nilai margin tertentu, akibatnya banyak element yang berantakan. Saya berusaha override di file CSS yang sudah saya buat, dengan menuliskan nilai margin = 0, tetapi pada kenyataannya tidak berubah, sehingga saya harus sercing. Saya pun menemukan solusinya di internet yakni dengan mengubah nilai properti yang ingin saya ubah, yakni Margin, menjadi "initial".

Selanjutnya adalah cara menyimpan data User ketika sudah login. Walaupun pak Eka tidak menyuruh saya dan mahasiswa lainnya untuk mengimplementasikan login layaknya login sungguhan, tetapi saya merasa ada beberapa mockup yang sudah saya buat namun hanya bisa tersambung jika login sudah terjadi. Saya memikirkan bagaimana cara menyimpan data tanpa harus menggunakan database ataupun cara lain yang melibatkan Back End, akhirnya saya terfikirkan untuk menggunakan Cookie. Yang jadi masalah adalah saya tidak tahu cara mengimplementasikan Cookie, saya pun memutuskan untuk melakukan Searching, dan menemukan jawabannya, yaitu menggunakan JavaScript "document.cookie = (data yang ingin disimpan)".

Yang terakhir adalah implementasi dari element Sticky. Saya sering menggunakan website, biasanya Facebook di mana pada website Facebook banyak sekali element Sticky. Sebagai informasi, element Sticky adalah sebutan untuk element yang posisinya tetap (fixed) relatif dengan layar pengguna, jadi walaupun pengguna melakukan Scrolling, element Sticky akan terus tampil mengikuti. Saya akhirnya melakukan Searching dan menemukan solusinya yaitu dengan menetapkan nilai Position = fixed dan melakukan sedikit pengaturan lainnya. Pada akhirnya saya berhasil mengimplementasikan Sticky pada navbar dan informasi user yang sudah login.

Walaupun tidak mudah mengimplementasikan hal-hal tersebut, saya bersyukur memiliki koneksi internet sehingga bisa menemukan solusinya.

Impossible to Achieve

Selain ada hal yang sulit diimplementasikan, ada hal lain yang ternyata tidak mungkin bagi saya untuk diimplementasikan. Sebenarnya ada beberapa yang tidak mungkin saya implementasikan, baik karena faktor waktu ataupun memang saya sudah buntu memikirkannya, tetapi karena pada Quiz ini hanya diminta satu, maka saya akan menyebutkan satu saja yang menurut saya tidak mungkin untuk diimplementasikan.

Bagian yang menurut saya paling tidak mungkin untuk diimplementasikan adalah fitur membuat job (Add Job). Saya bingung bagaimana bentuk form yang akan ditampilkan, karena user yang membuat harus memasukan gambar dari pekerjaan, menentukan pekerjaan (Task) apa yang harus dikerjakan, menentukan Reward, dll. Sangat sulit bagi saya untuk mengubah Task yang dibuat user yang merupakan kalimat deskriptif menjadi kuantitatif yang menjadi indikator selesainya pekerjaan. Saya pun akhirnya memutuskan untuk menghapus fitur tersebut dengan mengubah tombolnya menjadi Disabled.

Sebenarnya tidak tepat menyebut hal tersebut tidak mungkin (Impossible). Jika diberi waktu lebih, saya mungkin bisa memikirkan dan mengimplementasikannya, namun kembali lagi ke tujuan utama diberikannya tugas ini, yaitu melatih kemampuan saya dan teman-teman saya di bidang Front End, sehingga ada atau tidak adanya fitur tersebut tidak terlalu berpengaruh besar. Adapun untuk Mockup sebenarnya (Original Mockup) SUDAH SAYA IMPLEMENTASIKAN SEMUANYA, tetapi untuk bagian yang Impossible tidak saya masukkan ke Mockup dari awal, karena yang membuat hal tersebut Impossible bukanlah implementasi CSS dan HTML nya melainkan memikirkan bagaimana bentuknya, sehingga saya tidak membuat Mockup-nya.
